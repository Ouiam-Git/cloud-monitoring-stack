# Cloud Monitoring Stack

Automated cloud monitoring stack deployed with Terraform & Ansible, visualized with Grafana.

## Architecture

```
GitHub Actions
     в”Ӯ
     в”ңв”Җв”Җ Terraform в”Җв”Җв–ә AWS EC2 (t3.small, Ubuntu 22.04)
     в”Ӯ                      в”Ӯ
     в””в”Җв”Җ Ansible в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Ө
                            в”ңв”Җв”Җ Prometheus  :9090
                            в”ңв”Җв”Җ Node Exporter :9100
                            в””в”Җв”Җ Grafana     :3000
```

## Stack

| Tool | Role |
|------|------|
| **Terraform** | Provision EC2, Security Group, Elastic IP |
| **Ansible** | Install & configure Prometheus, Grafana, Node Exporter |
| **Prometheus** | Metrics collection & storage |
| **Node Exporter** | System metrics (CPU, Memory, Disk, Network) |
| **Grafana** | Visualization dashboard |
| **GitHub Actions** | CI/CD pipeline (plan вҶ’ apply вҶ’ provision) |

## Quick Start

### Prerequisites

- AWS account with credentials
- Terraform >= 1.3.0
- Ansible >= 9.0
- SSH key pair

### 1. Configure AWS credentials

```bash
export AWS_ACCESS_KEY_ID="your-key"
export AWS_SECRET_ACCESS_KEY="your-secret"
export AWS_DEFAULT_REGION="us-east-1"
```

### 2. Deploy with Terraform

```bash
cd terraform
terraform init
terraform plan
terraform apply
```

### 3. Provision with Ansible

```bash
cd ansible
ansible-playbook -i inventory/hosts.ini playbooks/monitoring.yml
```

### 4. Access dashboards

After deployment, Terraform outputs the URLs:

```
grafana_url    = "http://<PUBLIC_IP>:3000"
prometheus_url = "http://<PUBLIC_IP>:9090"
ssh_command    = "ssh -i ~/.ssh/id_rsa ubuntu@<PUBLIC_IP>"
```

- **Grafana**: `http://<PUBLIC_IP>:3000` вҶ’ login: `admin / admin`
- **Prometheus**: `http://<PUBLIC_IP>:9090`

## GitHub Actions Setup

Add these secrets in your GitHub repository (`Settings > Secrets`):

| Secret | Description |
|--------|-------------|
| `AWS_ACCESS_KEY_ID` | AWS access key |
| `AWS_SECRET_ACCESS_KEY` | AWS secret key |
| `AWS_REGION` | e.g. `us-east-1` |
| `SSH_PUBLIC_KEY` | Content of `~/.ssh/id_rsa.pub` |
| `SSH_PRIVATE_KEY` | Content of `~/.ssh/id_rsa` |

Push to `main` вҶ’ pipeline runs automatically:
1. Terraform validates & plans
2. Terraform applies (provisions EC2)
3. Ansible installs Prometheus + Grafana

## Dashboard Metrics

The Grafana dashboard monitors:

- **CPU**: Usage %, IO Wait % (gauge + time series)
- **Memory**: Used / Available / Total RAM
- **Network**: Bytes received / transmitted per interface
- **Disk**: Read / Write I/O + usage %

## Teardown

```bash
cd terraform
terraform destroy
```

Or trigger `workflow_dispatch` with action = `destroy` from GitHub Actions.
